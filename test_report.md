## 一、测试环境准备

### 1.1 前置条件

- Python 3.10+ 已安装
- Node.js 18+ 已安装
- 后端依赖已安装（`pip install -r requirements.txt`）
- 前端依赖已安装（`npm install`）
- 旧数据库 `knowq.db` 已删除（首次启动自动重建）

### 1.2 启动服务

```bash
# 终端1：启动后端
cd backend
python main.py

# 终端2：启动前端
cd frontend
npm run dev
```

## 三、Bug修复验证测试

### 3.1 BUG-01：qa_engine.py 中 db 应为 self.db

| 项目 | 内容 |
|------|------|
| Bug描述 | `_get_llm_gateway` 方法第52行使用了未定义的 `db` 变量，应为 `self.db` |
| 触发条件 | 系统未配置默认LLM且未指定llm_config_id时 |
| 验证步骤 | 1. 创建一个问答系统，不指定LLM配置 |
| | 2. 在LLM配置页面，确保没有配置被设为默认 |
| | 3. 向该问答系统发送问题 |
| 预期结果 | 系统自动选择任意一个LLM配置（不抛出NameError） |
| | 如果完全没有LLM配置，返回友好错误提示而非500错误 |

### 3.2 BUG-02：运算符优先级导致搜索逻辑错误

| 项目 | 内容 |
|------|------|
| Bug描述 | `(entry.question or "" + " " + entry.content)` 中 `"" + " " + entry.content` 先于 `or` 执行 |
| 触发条件 | entry.question 有值时，搜索文本不包含 entry.content |
| 验证步骤 | 1. 创建一个知识条目：question="如何请假"，content="请假需要提前3天在OA系统提交" |
| | 2. 使用AI模式搜索"OA系统" |
| 预期结果 | 该条目被检索到（修复前因content未参与搜索匹配，可能检索不到） |

### 3.3 BUG-03：source判断永远为"kb"

| 项目 | 内容 |
|------|------|
| Bug描述 | `source = "kb" if matched else "kb"` 两个分支返回相同值 |
| 验证步骤 | 1. 使用AI模式问答系统 |
| | 2. 发送一个与知识库完全无关的问题 |
| | 3. 查看问答日志 |
| 预期结果 | 日志中 source 字段为 "llm"（而非 "kb"） |
| | 发送知识库相关问题后，source 为 "kb" |

### 3.4 BUG-04：密码通过URL查询参数传输

| 项目 | 内容 |
|------|------|
| Bug描述 | 修改密码接口的参数通过URL查询参数传输，密码会出现在URL中 |
| 验证步骤 | 1. 打开浏览器开发者工具 Network 面板 |
| | 2. 修改密码 |
| | 3. 检查请求 |
| 预期结果 | 请求方法为 PUT，参数在 Request Body 中（JSON格式） |
| | URL中不包含密码参数 |

### 3.5 BUG-06：问答解析双冒号分割逻辑缺陷

| 项目 | 内容 |
|------|------|
| Bug描述 | `stripped.split(":", 1)[-1].split("：", 1)[-1]` 会在内容含冒号时错误截断 |

| 预期结果 | 解析出的问题为"什么是：KnowQ？"（完整保留冒号后的内容） |
| | 修复前会错误得到"KnowQ？" |

### 3.6 BUG-07：分块重叠参数导致无限循环

| 项目 | 内容 |
|------|------|
| Bug描述 | 当 chunk_overlap >= chunk_size 时，start 不会前进，导致无限循环 |

| 预期结果 | 导入正常完成，不卡死 |
| | 每块100字符，start每次至少前进1个字符 |

### 3.7 BUG-11：Token过期时间未使用SystemSettings

| 项目 | 内容 |
|------|------|
| Bug描述 | auth.py 中硬编码 ACCESS_TOKEN_EXPIRE_HOURS=24，不读取数据库配置 |
