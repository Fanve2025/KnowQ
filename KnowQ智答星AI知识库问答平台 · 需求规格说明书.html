KnowQ智答星AI知识库问答平台
需求规格说明书

文档版本 V1.0  |  修改日期：2026-04-28  |  修改人：系统组  |  修改说明：初始版本创建

1. 系统概述

1.1 项目背景

为组织提供一套可配置、可组合的智能问答系统。管理员可通过管理平台创建多个知识库，配置大模型与搜索策略，并一键生成面向最终用户的AI问答前端。系统支持AI模式（知识库+联网搜索）与规则模式（关键字智能匹配），实现灵活的知识服务。

1.2 产品范围

  • 管理平台（后台）：知识库管理、问答系统生成与管理、模型及搜索配置、数据统计、用户权限管理。
  • AI问答系统前端（用户端）：由管理平台生成的独立问答界面，普通用户通过专属地址访问并进行问答交互。

1.3 用户角色

角色 职责说明
管理员 创建/管理知识库；配置模型与搜索；生成/管理问答系统前端；查看统计；管理用户。
普通用户 访问指定的问答系统前端，提交问题进行AI问答。

2. 系统总体逻辑与模式说明

2.1 AI模式

由大模型（LLM）主导，分为两步判断：

  1. LLM意图识别：判断用户问题是否存在于关联知识库。
    • 存在：LLM基于知识库内容生成回复。
    • 不存在：调用Tavily联网搜索API获取互联网信息，将搜索结果返回LLM，由LLM整合生成自然语言回复。
  2. 该过程需将知识库内容作为上下文注入提示词（Prompt），并支持对话历史记录。
🔁 AI模式流程： 用户提问 → LLM判断知识库是否存在 → 存在：基于知识库生成回复 / 不存在：Tavily联网搜索 → LLM整合搜索结果 → 生成自然语言回复

2.2 规则模式

不依赖大模型，采用关键字匹配算法：

  1. 对用户输入进行分词、同义词/近义词扩展，与知识库条目进行智能匹配（如TF-IDF、BM25或相似度计算）。
  2. 匹配成功（达到阈值）：基于匹配到的知识库内容生成标准回复。
  3. 匹配失败：返回预设的智能提示（如"未找到相关内容，请尝试其他关键词"），可附带引导问题。

3. 管理平台功能需求

3.1 管理员登录与账号安全

  • 提供账号密码登录，可支持二次验证（可选）。
  • 密码加密存储，会话超时自动退出。

3.2 知识库管理

  • 创建知识库：填写名称、描述，创建空知识库。
  • 编辑知识库内容：
    • 支持手动添加/编辑/删除"问答对"（标准问题+答案）。
    • 支持批量导入文档（TXT、PDF、Word、Markdown等），系统自动解析入库。
    • 支持文档段落/分块管理，可设定分块大小。
  • 知识库列表：查看所有知识库，支持搜索、筛选。
  • 知识库组合：一个问答系统可选配一个或多个知识库（多库联合检索）。

3.3 AI问答系统生成与管理

  • 创建问答系统：
    • 输入系统名称、选择关联的一个或多个知识库。
    • 选择运行模式：AI模式 / 规则模式。
    • 系统自动生成唯一的前端访问地址（URL）。
  • 问答系统列表：集中展示已生成的系统，包括名称、关联知识库、模式、访问地址、创建时间。
  • 编辑/启停：可修改配置，可关闭某个前端使其不可访问。
  • 预览：提供内嵌预览窗口直接测试问答效果。

3.4 大模型配置（全局设置，亦可绑定到单个系统）

  • 默认LLM提供商下拉选项，内置如下： Anthropic、OpenAI、Google、xAI、DeepSeek、Kimi、智谱AI、Minimax、阿里云、硅基流动、无问芯穹、模力方舟、自定义。
  • 选择提供商后需填写：
    • API Key（加密存储）
    • 模型名称/版本
    • 自定义Endpoint（如适用）
    • 请求参数：温度、最大Token数、top_p等。
  • 支持测试连接功能，验证配置是否可用。
  • 选择"自定义"时，管理员手动输入模型名称、API地址、密钥格式等。

3.5 联网搜索配置

  • 配置Tavily Search API密钥（或其他可选搜索引擎，但默认指定Tavily）。
  • 开启/关闭联网搜索功能（可绑定到问答系统）。
  • 设置最大搜索条数、搜索结果摘要长度等。

3.6 数据统计看板

  • 总览统计：知识库总数、文档总数、问答系统数、注册用户数（如有）。
  • 单系统统计：各问答系统的访问量、提问次数、知识库命中率、联网搜索触发次数。
  • 时间维度筛选（今天/7天/30天/自定义）。
  • 可视化图表展示（柱状图、折线图）。

3.7 用户管理

  • 用户列表：显示用户名、角色（管理员/普通用户）、创建时间、状态。
  • 新增用户：管理员可创建账号，分配角色。
  • 权限控制：仅管理员角色能登录管理平台；普通用户仅能访问前端系统。
  • 密码重置、账号启用/禁用。

4. AI问答系统前端功能需求

4.1 用户交互界面

  • 简洁的聊天窗样式，支持问题输入框与发送按钮。
  • 对话历史显示（区分用户问题和AI回答）。
  • 支持输入框自动聚焦、Enter发送。
  • 加载状态提示（思考中、搜索中）。

4.2 回答呈现

  • AI模式下：回答底部可标注"基于知识库生成"或"基于联网搜索结果"。
  • 规则模式下：明确显示匹配的知识条目或提示未找到。
  • 支持引用来源（知识库文档段落链接或搜索网页链接）。

4.3 反馈机制（可选）

  • 用户可以点赞/点踩某条回答，数据回传管理平台用于优化。

5. 非功能性需求

类别 要求
性能 问答接口响应时间在知识库模式下≤3秒，联网搜索模式下≤8秒。
可用性 系统可用性≥99.5%，支持故障自动恢复。
安全性 所有API密钥加密存储；前端访问可设置密码或IP白名单（可选）；HTTPS强制。
扩展性 支持水平扩展知识库服务与LLM调用节点，新增提供商时仅需配置。
兼容性 管理平台兼容Chrome/Firefox/Edge最新版本；前端响应式适配移动端。
可维护性 管理操作均有操作日志记录；重要操作可回滚。

6. 数据模型（核心实体）

6.1 用户表 (users)

id, username, password_hash, role (admin/user), status, created_at

6.2 知识库表 (knowledge_bases)

id, name, description, created_by, created_at, updated_at

6.3 知识条目/文档段表 (knowledge_entries)

id, kb_id, type (q_a / document_chunk), question (用于问答对), content (答案或段落文本), source_doc, embedding_vector (可选，用于语义搜索), created_at

6.4 问答系统表 (qa_systems)

id, name, kb_ids (多对多关联), mode (ai/rule), llm_config_id, enable_web_search, frontend_url, status, created_at

6.5 LLM配置表 (llm_configs)

id, provider (枚举), model_name, api_key_encrypted, endpoint, params_json, is_default, created_at

6.6 联网搜索配置表 (search_configs)

id, api_key_encrypted, max_results, summary_length, is_enabled_global

6.7 问答日志表 (qa_logs)

id, system_id, user_id (可为空), question, answer, mode, source (kb/web/rule), matched_kb_id, feedback, created_at

7. 系统架构建议

  • 前端：管理平台使用Vue/React；问答前端支持独立部署或多租户动态加载。
  • 后端服务：微服务拆分
    • 知识库管理服务（内容向量化可选）
    • 问答引擎服务（集成LLM路由、联网搜索）
    • 用户与权限服务
    • 统计服务
  • 向量数据库（可选，用于AI模式语义搜索）：Milvus / Pinecone / pgvector。
  • 消息队列：文档解析入库与向量化可用异步队列。
  • LLM网关：统一适配不同提供商的API差异（适配器模式）。

8. 接口需求

8.1 管理端API（举例）

POST   /api/admin/kb              → 创建知识库
                    PUT    /api/admin/kb/{id}         → 更新知识库
                    POST   /api/admin/kb/{id}/entries → 批量导入条目
                    POST   /api/admin/systems         → 创建问答系统
                    GET    /api/admin/systems         → 问答系统列表
                    POST   /api/admin/llm-config      → 配置大模型
                    GET    /api/stats/dashboard       → 统计数据

8.2 前端问答API

POST /api/ask/{systemId}
                    请求体：{ "question": "...", "history": [...] }
                    响应：{ "answer": "...", "source": "kb"|"web"|"rule", "references": [...] }

9. 验收标准

  1. 管理员可成功创建知识库并导入至少1000条问答对，解析无明显错误。
  2. 配置OpenAI/DeepSeek等至少三种提供商，测试连接正常。
  3. 创建一个使用AI模式的问答系统，分别询问库内和库外问题，验证LLM知识与联网搜索切换无误。
  4. 规则模式下，准确匹配阈值内的关键字并返回答案，无匹配时返回友好提示。
  5. 生成的前端链接可独立访问，并完成完整问答流程。
  6. 统计看板数据实时更新，与实际操作一致。

此说明书可作为开发与测试的基础，若有细节需要深化，可在详细设计阶段补充。
